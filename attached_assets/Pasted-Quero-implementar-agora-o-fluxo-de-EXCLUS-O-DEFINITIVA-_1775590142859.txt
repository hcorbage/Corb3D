FLUXO OBRIGATÓRIO
Ao excluir uma empresa, o sistema deve executar nesta ordem:

1. Validar permissões (super_admin apenas)
2. Validar confirmação forte no frontend e backend
3. Gerar backup por empresa
4. Gerar backup global administrativo
5. Excluir os dados da empresa
6. Excluir a conta principal da empresa (company_admin)
7. Registrar tudo em audit_logs
8. Responder com sucesso contendo os nomes dos backups gerados

BACKUP POR EMPRESA
Usar o sistema de backup por empresa já existente.

BACKUP GLOBAL ADMINISTRATIVO
Quero implementar um backup global administrativo separado, para permitir recuperar depois contas e estrutura global necessárias.

Salvar em pasta separada:
`/var/backups/corb3dapp/admin/`

Formato:
`admin_<timestamp>.json.gz`

Esse backup global deve conter tudo o que for necessário para restaurar a identidade global da empresa excluída depois, incluindo:
- users
- user_permissions
- password_reset_tokens (se fizer sentido)
- audit_logs (avaliar se entra ou não)
- e qualquer outro dado global essencial

Quero que você analise com precisão quais tabelas globais precisam entrar nesse backup administrativo.

Como o sistema usa users.id do company_admin como chave de isolamento, quero que você trate isso com muito cuidado.
